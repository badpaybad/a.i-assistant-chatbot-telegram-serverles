# Phase 1: Thinking 🧐
> **Mục tiêu:** Hiểu rõ "Tại sao" và "Cái gì".

- [ ] **Yêu cầu:** Tóm tắt yêu cầu người dùng bằng ngôn ngữ của bạn.
- [ ] **Phân tích Context:** Sử dụng `@file` để liệt kê các file liên quan nhất.
- [ ] **Ràng buộc:** Có giới hạn nào về công nghệ, thư viện hoặc hiệu suất không?
- [ ] **Đầu ra mong muốn:** Định nghĩa rõ kết quả cuối cùng trông như thế nào.

# Phase 2: Reasoning 🧠
> **Mục tiêu:** Xây dựng logic trước khi gõ phím.

- [ ] **Luồng dữ liệu:** Dữ liệu đi từ đâu đến đâu?
- [ ] **Giải pháp kỹ thuật:** - Phương án A (Ưu/Nhược): ...
    - Phương án B (Ưu/Nhược): ...
- [ ] **Lựa chọn:** Tại sao chọn phương án này?
- [ ] **Dự đoán lỗi:** Liệt kê các Edge Cases (Lỗi mạng, dữ liệu rỗng, type mismatch).

# Phase 3: Action 🛠️
> **Mục tiêu:** Triển khai code sạch và chuẩn.

- [ ] **Cấu trúc:** Tạo/Sửa các hàm, Class cần thiết.
- [ ] **Style Guide:** Kiểm tra naming convention (camelCase, PascalCase...).
- [ ] **Comments:** Thêm JSDoc hoặc Python Docstrings cho các hàm phức tạp.
- [ ] **Refactor:** Tối ưu hóa các đoạn mã lặp lại (DRY principle).

# Phase 4: Verify ✅
> **Mục tiêu:** Đảm bảo mọi thứ chạy đúng như thiết kế.

- [ ] **Unit Test:** Viết hoặc chạy các test case cơ bản.
- [ ] **AI Code Review:** Dùng prompt: `Review this code for security and performance`.
- [ ] **Manual Check:** Thử nghiệm các trường hợp biên đã liệt kê ở bước 2.
- [ ] **Logs/Debug:** Kiểm tra terminal/console để đảm bảo không có warning "ngầm".

# Phase 5: Finalize 🏁
> **Mục tiêu:** Đóng gói và bàn giao.

- [ ] **Cleanup:** Xóa code thừa, TODO notes, console logs.
- [ ] **Documentation:** Cập nhật README hoặc API docs nếu có thay đổi interface.
- [ ] **Commit Message:** Ghi lại thay đổi theo chuẩn (ví dụ: `feat: add oolong tea selection logic`).
- [ ] **Done:** Xác nhận hoàn thành task.
